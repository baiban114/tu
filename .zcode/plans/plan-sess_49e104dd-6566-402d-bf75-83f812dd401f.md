# 系统设置页面重构计划

## 目标
把「系统配置」页重构为「系统设置」：左侧 `el-menu` 竖向列出功能模块（**编辑器**、**AI**），右侧显示对应模块。
- **AI 模块** = 现有 AI Agent API 配置（迁出为独立子组件，逻辑不变，仍走后端 `/api/ai/settings`）。
- **编辑器模块** = 新增「文本选择后出现的工具栏」开关（`el-switch`），通过新建后端 API 持久化（含 mock 分支）。
- 布局与组件优先用已有 Element Plus；遵循 AGENTS.md 的页面滚动约束与 region-scoped scroll 约定。

---

## 一、后端（tu-backend）— 新增 editor preferences 持久化

复刻现有 AI settings 的「全局单记录」成熟模式（`CONFIG_ID="default"`，Hibernate `ddl-auto=update` 自动建表，无需 SQL 迁移文件）。

新建 feature 包 `com.tu.backend.editorpreferences`（含标准子包 controller/service/dto/entity/repository）：

| 新文件 | 说明 |
|------|------|
| `entity/EditorPreferencesEntity.java` | `@Table(name="editor_preferences")`，`@Id String id="default"`，`boolean selectionToolbarEnabled`（默认 true），`createdAt/updatedAt` + `@PrePersist/@PreUpdate`（照搬 `AiAgentConfigEntity`） |
| `repository/EditorPreferencesRepository.java` | `JpaRepository<EditorPreferencesEntity, String>` |
| `dto/EditorPreferencesDto.java` | `record EditorPreferencesDto(boolean selectionToolbarEnabled)` |
| `dto/UpdateEditorPreferencesRequest.java` | `record UpdateEditorPreferencesRequest(boolean selectionToolbarEnabled)` |
| `service/EditorPreferencesService.java` | `getPreferences()` 读取或返回默认 DTO（默认 `selectionToolbarEnabled=true`，保持现有行为）；`updatePreferences(req)` load-or-create + save；`@Transactional(readOnly=true)` 读 / `@Transactional` 写 |
| `controller/EditorPreferencesController.java` | `@RestController @RequestMapping("/api/editor-preferences")`；`GET` → `ApiResponse<EditorPreferencesDto>`；`PUT @Valid` → 更新并返回 DTO |

默认值 = **true**（工具栏默认开），保证现有用户无感知，直至手动关闭。

> 说明：沿用全局单记录（与 AI 配置一致，当前 `SecurityConfig.permitAll` 无 per-user 鉴权在用）。如后续需要 per-user，可再加 `user_id` 列，本次不做。

## 二、前端（tu-web-ts）

### 1. 新增 API 模块（backend/mock 双分支，照搬 `aiSettings.ts`）
**`src/api/editorPreferences.ts`**
- 类型 `EditorPreferences { selectionToolbarEnabled: boolean }`、`UpdateEditorPreferencesPayload`
- `getEditorPreferences()` → backend: `GET /api/editor-preferences`；mock: 读 localStorage `tu:mock-editor-preferences`（默认 true，`try/catch` 回退）
- `updateEditorPreferences(payload)` → backend: `PUT`；mock: 写 localStorage 后返回
- mock 分支返回裸数据（与 `request()` 解包行为一致）

### 2. 新增 Pinia store（跨组件共享：TuEditorPage 读、SettingsView 读写）
**`src/stores/editorPreferences.ts`**
- `selectionToolbarEnabled` (ref, 默认 true)、`loaded` (ref)
- `load()`：幂等（`loaded` 判定），调用 `getEditorPreferences()`
- `setSelectionToolbarEnabled(v)`：更新本地状态 + 调 `updateEditorPreferences()`（出错时 `el-message` 提示并回滚状态）

### 3. 重构 SettingsView.vue 为「系统设置」外壳
**`src/views/SettingsView.vue`**（重写为薄外壳）
- 顶部 header：「系统设置」标题 + 「返回工作区」链接（route 保持 `/settings` 不变）
- 布局：`height:100vh; display:flex; flex-direction:column`；header 固定；下方 `flex:1; min-height:0` 行容器：
  - **左**：`<el-menu>` 竖向（`:default-active` 绑定 `activeMenu`，`@select` 切换），菜单项「编辑器」「AI」（带 `el-icon`）
  - **右**：`flex:1; overflow-y:auto` 内容区，按 `activeMenu` 用 `v-if` 渲染 `EditorSettingsPanel` / `AiSettingsPanel`
- 全部用 Element Plus（`el-menu`、`el-menu-item`、`el-icon`），符合「优先用已有 elementplus」

### 4. 抽取现有 AI 配置为子组件（逻辑零改动，纯搬运）
**`src/components/settings/AiSettingsPanel.vue`**
- 把 SettingsView 现有的 AI 配置表单 + Agent 记录表格（含 `aiSettings`/`aiRuns` API 调用、`form` 状态、`saveSettings/testConnection/loadRunLogs` 等函数与相关 scoped 样式）整体迁入
- API/行为不变，仍是后端 `/api/ai/settings` + `/api/ai/runs`

### 5. 新增编辑器配置子组件
**`src/components/settings/EditorSettingsPanel.vue`**
- `<el-form label-position="top">` + `<el-form-item>` + `<el-switch>` 控制「文本选择后出现的工具栏」
- 绑定 `useEditorPreferencesStore`；`onMounted` 调 `store.load()`；switch change → `store.setSelectionToolbarEnabled(v)`
- 说明文案：关闭后编辑器内选中文本不再弹出工具栏

### 6. 接线 TuEditorPage.vue（应用开关）
**`src/components/TuEditorPage.vue`**
- 引入 `useEditorPreferencesStore`，`onMounted` 调 `load()`
- 修改 577-585 行 `selectionToolbarSuppressed` 计算属性 → 增加 `|| !editorPreferencesStore.selectionToolbarEnabled`（即关闭开关时等同 suppress）
- 这样开关即时生效（store 响应式驱动 `:suppressed`）

### 7. 文案与文档同步
- **`src/views/HomeView.vue:164`**：链接文案「系统配置」→「系统设置」（route 不变）
- **`src/components/MultiTableBlock.vue:621`**：错误提示「系统配置」→「系统设置」（保持一致）
- 更新相关 docs/README 中对「系统配置」的称呼（如 `tu-web-ts` 内文档有提及）
